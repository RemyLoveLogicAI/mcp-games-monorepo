## Overview

The apps directory is a container for application-level source trees in this repository. At the root of this directory there are no files; instead it organizes one or more application subdirectories. Currently it hosts the omnigentic application area which contains its own TypeScript source tree under a src/ root. The omnigentic subtree is described as having a src/ directory with a primary index.ts at its root and related implementation subtrees organized beneath that.

This directory's role is organizational: it groups independently developed application areas so developers can find the app-specific source in predictable places (for example apps/omnigentic/src). Build, development, and review activities for an application should be performed inside the corresponding subdirectory. Because the apps directory has no root-level files, contributors should treat each subdirectory as the effective code root for that application area and follow the structure and entry points defined therein.


### File Organization

This directory uses a hierarchical layout where each application gets its own subdirectory. Root-level files are absent; application code resides under each subdirectory's src/ root (for example omnigentic/src). This keeps app code isolated and makes the apps directory a registry of application areas rather than a place for shared code.
